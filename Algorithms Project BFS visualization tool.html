Why O(V + E)?

Each vertex is enqueued exactly once when first discovered, dequeued exactly once — O(V) queue operations. Each edge is examined at most twice in an undirected graph — O(E) checks. Total: O(V + E).

  • Dense graph (E ≈ V²): effectively O(V²)
  • Sparse graph (E ≈ V): effectively O(V)
  • Linear in the size of the graph representation

BFS vs DFS — The Critical Difference

Same time and space complexity O(V + E), fundamentally different search order. BFS uses a FIFO queue, exploring all nodes at distance d before any at d+1 — guaranteeing shortest paths. DFS uses a LIFO stack, diving deep first. For unweighted shortest paths, always use BFS.

BFS as a Greedy Algorithm

BFS embodies the greedy paradigm: at every step, it makes the locally optimal choice — always process the node with the smallest known distance first. Because all edges have equal weight in an unweighted graph, the FIFO queue naturally enforces this. Nodes at distance d are fully processed before any at d+1.

  • Greedy choice: always expand the closest undiscovered node
  • This choice is globally optimal — proven by induction
  • No revisiting needed — the first discovery is always optimal
  • Dijkstra generalizes this with a priority queue for weights

The Optimality Proof

Claim: When BFS first discovers v, dist[v] = δ(s, v) — the true shortest distance.

Proof by induction. Base: dist[s] = 0 = δ(s, s). ✓ Inductive step: assume all nodes at distance < k have correct dist. When u at distance k−1 is dequeued, for each white neighbor v: dist[v] = dist[u]+1 = k. No shorter path to v exists through any unprocessed node — all have distance ≥ k. So dist[v] = k = δ(s, v). ✓

Key insight: The FIFO queue guarantees processing in non-decreasing distance order — exactly what the greedy proof requires.

The Wave Analogy

Think of BFS as a stone dropped in still water. The ripple expands level by level. Each level corresponds to one unit of distance from source. The wavefront (GRAY nodes) separates discovered from undiscovered territory.

When Greedy BFS Fails

BFS fails for shortest paths when edges have different weights — it treats all edges as cost 1. Dijkstra fixes this by replacing the FIFO queue with a min-heap, preserving the greedy invariant for arbitrary non-negative weights.

  • Unweighted → BFS, O(V+E), simple queue
  • Non-negative weights → Dijkstra, O((V+E) log V), priority queue
  • Negative weights → Bellman-Ford, O(VE), DP relaxation

FIFO Queue Invariant

The queue always holds distances [d, d+1]. Nodes at d are processed before d+1. This is why BFS finds shortest paths.

Gray vs Black

GRAY = frontier — discovered, neighbors not yet explored. BLACK = fully done. WHITE = undiscovered. Three colors enable cycle detection (CLRS).

Disconnected Graphs

BFS only reaches the same connected component as source. For disconnected graphs, run BFS once per unvisited component.
